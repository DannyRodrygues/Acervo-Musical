Bem-vindo ao Acervo Musical

Olá! Meu nome é Daniella Oliveira Rodrigues, sou graduada em Análise e Desenvolvimento de Software e estou cursando Pós-Graduação em Especialização em Java pela UTFPR. Este projeto foi criado como parte dos meus estudos na área de Banco de Dados e no desenvolvimento de APIs Restful.

Objetivo

A API de acervo musical foi desenvolvida para fins educacionais, utilizando a linguagem Java e o ambiente de desenvolvimento IntelliJ. Objetivo é apresentar a API sendo consumida e disponibilizada via WEB nesta pagina, e informar como foi realizado o projeto.

Estrutura do Projeto

O projeto segue uma arquitetura em camadas, com separação clara entre entidade, repositório, serviço e controlador. O projeto está dividido em camadas bem definidas, como mostrado abaixo:

  • Entity: Representa as entidades do sistema, como "Cantor". A classe Cantor, por exemplo, define os atributos de um cantor e usa anotações do Jakarta Persistence API (JPA) para mapeamento da tabela no banco de dados. Exemplo de classe:
  • Repository: A camada de repositório se comunica diretamente com o banco de dados usando o Spring Data JPA. O CantorRepository facilita operações CRUD (Create, Read, Update, Delete) na entidade "Cantor". Exemplo de interface:
  • Service: Nesta camada, a lógica de negócios é implementada. O CantorService possui métodos como listar, encontrar, salvar e deletar cantores. Ele também realiza as operações de CRUD utilizando o repositório. Exemplo de classe:
  • Controller: A camada de controle expõe os recursos da API através de endpoints HTTP. O CantorController gerencia requisições de GET, POST, PUT e DELETE para manipular cantores. Exemplo de classe:

Testes da API

Para testar os endpoints da API, foi utilizado o Postman, uma ferramenta que permite simular requisições HTTP, validar respostas e testar a API de maneira eficiente. Este projeto foi fundamental para consolidar meu aprendizado em Java, Spring Boot, e bancos de dados relacionais com MariaDB.
